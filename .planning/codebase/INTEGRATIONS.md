# External Integrations

Last updated: 2026-07-19

## External APIs

### Wikipedia (read-only)
- **Purpose**: Scrape SoC data tables for 13+ vendors
- **Source**: `src/soc_db/scraper_wikipedia.py`, `scripts/scraper_wikipedia.py`
- **Method**: `fetch()` in `src/soc_db/common.py` — HTTP GET with caching
- **Rate limiting**: 1-second sleep between requests via `time.sleep(1)` in `fetch()`
- **Cache**: Filesystem cache at `CACHE_DIR` (default: `/tmp/soc-db-cache/`), keyed by URL MD5 hash, TTL 86400s (24h)
- **User-agent**: `SOC-DB/1.0 (+https://github.com/vitkuz573/soc-db)`

### Apple Silicon (read-only)
- **Purpose**: Scrape Apple SoC data from Wikipedia
- **Source**: `src/soc_db/scraper_apple.py`, `scripts/scraper_apple.py`
- **Method**: Same `fetch()` caching + rate limiting as Wikipedia scraper

### Wikidata (read-only) — legacy/scripts
- **Purpose**: SPARQL queries for fringe vendors
- **Source**: `scripts/scraper_wikidata_sparql.py`, `scripts/scraper_wikidata.py`
- **Status**: Located in `scripts/` (legacy directory)

### Linux Device Tree (read-only) — legacy/scripts
- **Purpose**: Extract SoC data from Linux kernel DT compatible strings
- **Source**: `scripts/scraper_linux_dt.py`
- **Status**: Located in `scripts/` (legacy directory)

## Data Storage

### Filesystem — JSON Files
- **Format**: Plain JSON arrays, one file per vendor
- **Location**: `data/*.json` (44 files total)
- **Index**: `data/index.json` — auto-generated by `tests/validate.py`
- **Schema**: `schema/chip-schema.json` (JSON Schema draft-07)
- **Access pattern**: Loaded into memory on server start, TTL-based cache invalidation (300s default)
- **No database**: The project deliberately uses flat JSON files — no PostgreSQL, MySQL, SQLite, or other RDBMS

## Authentication / Auth Providers

- **API Key (optional)**: X-API-Key header validation via middleware (`api/main.py:180-187`)
- **Configured via**: `SOC_DB_API_KEY` environment variable
- **Default**: No authentication (key is `None`)

## Deployment

### Docker
- **Dockerfile**: Multi-stage build (builder + runtime), `python:3.12-slim` base
- **docker-compose.yml**: Single service with volume mount for `data/`, healthcheck, restart policy
- **Port**: 8000 (internal + external)
- **Workers**: 2 uvicorn workers in production

### Systemd Timer (auto-update)
- **Service**: `deploy/soc-db-update.service` — runs `auto-deploy.sh --skip-scrape`
- **Timer**: `deploy/soc-db-update.timer` — systemd timer for periodic updates
- **User**: `vitaly`
- **Working directory**: `/tmp/soc-db`

## Web UI

- **HTML file**: `index.html` (single-page app, 23347 bytes)
- **Swagger UI**: `docs/swagger.html`

## Monitoring / Observability

- **Health endpoint**: `GET /health` — liveness probe (returns 200/503)
- **Metrics endpoint**: `GET /metrics` — uptime, request count, RPS, cached chips, rate-limit clients
- **Structured logging**: JSON-formatted logs to stderr (configurable via `SOC_DB_LOG_LEVEL` and `SOC_DB_LOG_FORMAT`)
- **Request tracking**: `X-Request-ID` header on all responses
- **Rate limiting**: Sliding-window per-IP limiter (default: 100 req/60s)

## No External Dependencies (notable absences)

- No database system (no PostgreSQL, MySQL, Redis, etc.)
- No message queue
- No cloud provider SDKs
- No monitoring agents (Prometheus, Datadog, etc.)
- No CDN
- No OAuth/OIDC providers
