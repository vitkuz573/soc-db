    #### d) `setup_telemetry()` function

    ```python
    def setup_telemetry() -> None:
        """Initialise OpenTelemetry tracing and Prometheus metrics.

        Must be called once at application startup (from FastAPI lifespan).
        Idempotent on subsequent calls — internal guard prevents double init.

        Configures:
        - Tracer provider with service name and version
        - FastAPI auto-instrumentation (applied later via ``instrument_app``)
        - Prometheus metrics registry
        - Meter provider for custom metrics
        """
        if _setup_done:
            logger.warning("setup_telemetry called more than once — skipping")
            return

        # --- Resource with service identity ---
        resource = Resource.create({
            "service.name": settings.otel_service_name,
            "service.version": __version__,
        })

        # --- Tracer provider ---
        tracer_provider = TracerProvider(resource=resource)
        # Use ConsoleSpanExporter for development; replace with OTLP exporter in v2.2+
        span_processor = BatchSpanProcessor(ConsoleSpanExporter())
        tracer_provider.add_span_processor(span_processor)
        otel_trace.set_tracer_provider(tracer_provider)

        # --- Meter provider ---
        meter_provider = MeterProvider(resource=resource)
        otel_metrics.set_meter_provider(meter_provider)

        # --- Mark as initialised ---
        _setup_done = True

        logger.info(
            "Telemetry initialised",
            extra={
                "service_name": settings.otel_service_name,
                "version": __version__,
            },
        )
    ```

    Implementation notes:
    - `_setup_done` is a module-level boolean flag that prevents double initialisation (important because lifespan could be called in tests that reset app.state).
    - `ConsoleSpanExporter` writes spans to stderr as JSON — this is verbose but maximally visible during development. In production, spans go to stderr where they're captured by the container runtime; in v2.2+, an OTLP exporter replaces this.
    - `BatchSpanProcessor` batches spans for efficiency (default: export every 5 seconds or 512 spans, whichever comes first).
    - The `Resource` object carries service identity on every span — essential for trace correlation in distributed systems.

    #### e) `instrument_app()` function

    ```python
    def instrument_app(app: Any) -> None:
        """Apply FastAPI auto-instrumentation to a FastAPI application.

        Must be called after ``setup_telemetry()`` and after the FastAPI app
        is fully configured (all routes registered).  The ``/health`` endpoint
        is excluded from tracing to avoid noise.

        Args:
            app: A FastAPI application instance.
        """
        if not _setup_done:
            raise RuntimeError("setup_telemetry() must be called before instrument_app()")

        FastAPIInstrumentor.instrument_app(
            app,
            excluded_urls="/health",
            tracer_provider=otel_trace.get_tracer_provider(),
            meter_provider=otel_metrics.get_meter_provider(),
        )
        logger.info("FastAPI instrumentation applied — /health excluded from tracing")
    ```

    Key details:
    - `excluded_urls="/health"` — the string is passed to the instrumentation's server request hook; it's a comma-separated list of URL patterns. This causes the FastAPI instrumentation to skip creating spans for `/health` requests.
    - `tracer_provider` and `meter_provider` are explicitly passed to ensure they use the ones we configured.
    - The function is separate from `setup_telemetry()` because the FastAPI app must be fully constructed (all routes registered) before instrumentation is applied. In the lifespan flow, `setup_telemetry()` runs first, then routes are registered, then `instrument_app(app)` is called.

    #### f) `update_vendor_gauge()` function

    ```python
    def update_vendor_gauge(chips: list[dict[str, Any]]) -> None:
        """Update the vendor distribution Prometheus gauge from chip data.

        Resets all existing vendor label values, then sets new counts.
        Call this after chip data is loaded (from lifespan) and on refresh.

        Args:
            chips: List of chip dicts, each with a ``vendor`` key.
        """
        vendor_distribution.clear()
        counts: dict[str, int] = {}
        for c in chips:
            v = c.get("vendor", "Unknown")
            counts[v] = counts.get(v, 0) + 1
        for vendor_name, count in counts.items():
            vendor_distribution.labels(vendor=vendor_name).set(count)
        chips_total.set(len(chips))
        logger.debug("Vendor distribution gauge updated — %d vendors, %d chips", len(counts), len(chips))
    ```

    Note: `vendor_distribution.clear()` resets all time series for this gauge. This is safe because we call it before re-setting all values. Without the clear, stale vendor labels would persist between calls.
