       ### `RateLimiter` Protocol
       A `@runtime_checkable` Protocol class with:
       - `async def check(self, key: str) -> tuple[bool, int, int, float]` — returns (allowed, limit, remaining, reset_timestamp). `reset_timestamp` is a Unix epoch float representing when the current window expires.
       - `@property` `is_redis_connected(self) -> bool` — whether Redis is currently connected.
       - `@property` `active_clients(self) -> int` — approximate count of distinct tracked keys (default 0, meaningful for InMemory).

       ### `InMemoryRateLimiter`
       - Constructor: `__init__(self, limit: int, window: int)` — defaults from settings.
       - Internal state:
         - `_buckets: dict[str, list[float]]` using `defaultdict(list)`
         - `_lock = asyncio.Lock()`
         - `_limit`, `_window` from constructor
       - `async def check(self, key)`: per D-06 (refactored current pattern):
         1. `now = time.monotonic()`, `cutoff = now - self._window`
         2. Acquire lock
         3. Prune entries where `bucket[0] < cutoff` via `while bucket and bucket[0] < cutoff: bucket.pop(0)`
         4. `count = len(bucket)`, `remaining = max(0, self._limit - count)`, `allowed = remaining > 0`
         5. If allowed: `bucket.append(now)`
         6. `reset_time = cutoff + self._window` (time when current window fully resets)
         7. Release lock
         8. Return `(allowed, self._limit, remaining, reset_time)`
       - `is_redis_connected` property: return `False`
       - `active_clients` property: return `len(self._buckets)` (note: includes expired-key dict entries but acceptable approximation; add docstring noting it's approximate)

       ### `RedisRateLimiter`
       - Constructor: `__init__(self, redis_url: str, limit: int, window: int, socket_connect_timeout: int = 2)`
       - Internal state: `_redis` (lazy `None` → `redis.asyncio.Redis`), `_connected: bool = False`
       - `async def _ensure_connected(self)` — lazy connect:
         - If `self._redis is None`:
           - `import redis.asyncio as aioredis` (lazy import so redis-py is optional at module level)
           - `self._redis = aioredis.from_url(self._redis_url, socket_connect_timeout=self._socket_connect_timeout, single_connection_client=True)`
           - Try `await self._redis.ping()`; on success set `self._connected = True`
           - On any exception: log warning, set `self._redis = None`, set `self._connected = False`, raise `ConnectionError`
       - `async def check(self, key)` — sliding window via sorted sets:
         1. Ensure connected (catch `ConnectionError` → re-raise wrapped)
         2. `now = time.time()`, `window_start = now - self._window`, `redis_key = f"ratelimit:{key}"`
         3. **Prune + Count pipeline:**
            ```python
            pipe = self._redis.pipeline()
            pipe.zremrangebyscore(redis_key, 0, window_start)
            pipe.zcard(redis_key)
            results = await pipe.execute()
            count = results[1]
            ```
         4. If `count >= self._limit`:
            - `remaining = 0`, `reset_time = now + self._window`
            - Return `(False, self._limit, remaining, reset_time)`
         5. If `count < self._limit`:
            - `remaining = self._limit - count - 1`
            - **Record + TTL pipeline:**
              ```python
              pipe = self._redis.pipeline()
              pipe.zadd(redis_key, {str(now): now})
              pipe.expire(redis_key, self._window * 2)  # 2x window TTL for safety margin
              await pipe.execute()
              ```
            - `reset_time = now + self._window`
            - Return `(True, self._limit, remaining, reset_time)`
         6. On any redis exception: set `self._redis = None` (force reconnect next time), set `self._connected = False`, log warning, raise `ConnectionError`
       - `is_redis_connected` property: return `self._connected`
       - `active_clients` property: return 0 (not meaningful for Redis — would need SCAN)
       - `async def close(self)`: if `self._redis` is not None, `await self._redis.aclose()`; set to None

       ### `create_rate_limiter` Factory
       ```python
       async def create_rate_limiter(
           redis_url: str | None = None,
           limit: int = 100,
           window: int = 60,
           socket_connect_timeout: int = 2,
       ) -> RateLimiter:
       ```
       - If `redis_url` is None: log "In-memory rate limiter active" at INFO, return `InMemoryRateLimiter(limit, window)`
       - If `redis_url` is set:
         - Create `limiter = RedisRateLimiter(redis_url, limit, window, socket_connect_timeout)`
         - Try `await limiter._ensure_connected()`; if success, log "Redis rate limiter active" at INFO, return limiter
         - On `ConnectionError`: log warning "Redis unavailable at {redis_url}, falling back to in-memory rate limiter", return `InMemoryRateLimiter(limit, window)`
       - Do NOT let any exception propagate — always return a working RateLimiter

       ### Module-level docs
       - Docstring explaining module purpose, the two implementations, and the factory pattern.
       - Use type annotations throughout per mypy strict mode.
