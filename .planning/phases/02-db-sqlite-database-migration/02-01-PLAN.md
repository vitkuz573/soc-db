    ```python
    def migrate(db_path: Path | None = None, force: bool = False) -> dict:
        """Migrate all JSON vendor files to SQLite.
        
        1. Resolve db_path (default from settings.db_path)
        2. If force=True, call drop_tables() then create_tables()
        3. If not force and DB exists with chips, skip (no-op)
        4. Open connection, begin transaction
        5. For each vendor file in sorted(DATA_DIR.glob('*.json')) skipping index.json:
           a. Read JSON file
           b. For each chip dict, map fields to SQLite columns
           c. Execute INSERT OR REPLACE into chips table
           d. Track per-vendor counts
        6. Commit transaction
        7. Call rebuild_fts(conn) to populate FTS5 index
        8. Return dict with stats: {total_chips, per_vendor: {vendor: count}, db_path: str}
        
        Field mapping logic (chip_dict → SQL columns):
        - Simple scalar fields: direct assignment
        - list fields (aliases, devices, alternative_names, tags, gpu_api): json.dumps()
        - dict fields (cache, rating, benchmarks, sources): json.dumps()
        - date fields (announced, updated): use str() if date else None (stored as ISO text)
        - All other fields: chip.get(key, None) — NULL if not present
        - Exclude: None values stay as NULL in SQLite (no INSERT column) to match JSON behavior
        """

    def validate_migration(db_path: Path | None = None) -> dict:
        """Validate SQLite data against JSON source.
        
        Loads all chips from JSON (via common.load_all()) and from SQLite (via queries.get_all()).
        Compares every field of every chip:
          - Check total count matches
          - Check each chip by id exists in both
          - Check each field matches value (handling JSON serialization for lists/dicts)
        Returns dict with: {pass: bool, total: int, mismatches: [{id, field, expected, got}], missing_ids: [], extra_ids: []}
        """
        
    def ensure_migrated(db_path: Path | None = None) -> bool:
        """Ensure database is migrated. If not, run migration. Returns True if ready.
        Used by query functions on first call when use_json=False and DB doesn't exist."""
    ```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSON files → Migration | Unvalidated data from external scrapers enters SQLite |
| CLI user → SQLite | Local CLI invocation can trigger migration or queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-DB-01 | Spoofing | FTS5 MATCH query | medium | mitigate | Parameterized queries only — no string concatenation. FTS5 syntax errors caught and fall back to LIKE. |
| T-DB-02 | Tampering | Migration writes | medium | mitigate | Single transaction wraps entire migration — if any write fails, atomic rollback prevents partial state. validate_migration() verifies counts post-migration. |
| T-DB-03 | Information Disclosure | Local SQLite file | low | accept | Database is local-only with no network exposure. Schema contains only public SoC data (no PII/secrets). |
| T-DB-04 | Denial of Service | Migration | low | accept | Force flag drops and recreates; user-initiated CLI command, not automated. |
| T-DB-05 | Tampering | Config env vars | low | mitigate | Settings use Pydantic BaseSettings with env_prefix SOC_DB_. Values read-only after app start. |
| T-DB-SC | Tampering | sqlite3 (stdlib) | high | mitigate | stdlib sqlite3 is CPython built-in (no pip install). Treat as [ASSUMED] — no package legitimacy gate needed. |

</threat_model>
