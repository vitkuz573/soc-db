<objective>
Create the UUID-based canonical chip identity system and multi-strategy matcher.

**Purpose:** Every chip entering the database gets a deterministic UUID derived from (vendor + model-number fingerprint), independently of the name-derived slug. The multi-strategy matcher resolves duplicates by trying exact model → alias registry → vendor+model regex → Wikidata QID → fuzzy (rapidfuzz) fallback. The existing `write_vendor_file()` is updated to use the DedupEngine for all matching and dedup decisions.

**Output:** `src/soc_db/dedup.py` (new module), `src/soc_db/common.py` (modified write_vendor_file and _match_existing)
</objective>

  <action>
    Create `src/soc_db/dedup.py` with:

    1. **Imports:** `uuid` (stdlib), `logging`, `re`, `typing`, `rapidfuzz.fuzz` and `rapidfuzz.utils`. Import `VENDOR_KNOWLEDGE` from `soc_db.enrich._vendor_data` for vendor-model patterns. Import `extract_model` from `soc_db.common` for regex model extraction. Import `slug` from `soc_db.common` for fallback ID generation.

    2. `chip_uuid(vendor: str, model: str) -> str` — Generate UUID5 from namespace UUID `ns=uuid.UUID("6ba7b810-9dad-11d1-80b4-00c04fd430c8")` (DNS namespace) keyed on `f"{vendor.strip().lower()}:{model.strip().lower()}"`. Return result as hex string (str, 32 chars, no hyphens). If model is empty or None, use vendor + name as fallback.

    3. `CHIP_ALIASES: dict[str, list[str]]` — Module-level alias registry mapping normalized model keys to known alternative names. Populate with codename aliases extracted from existing chip data patterns (e.g., `"sm8550" -> ["kalama", "snapdragon 8 gen 2"]`, `"sm8650" -> ["pineapple", "snapdragon 8 gen 3"]`, `"mt6983" -> ["dimensity 9000"]`, `"t8103" -> ["apple m1", "apl1102"]`). This is a curated list — start with ~20 well-known mappings, designed to be extended by scrapers and administrators.

    4. `DedupEngine` class:
       - `__init__(self, alias_registry: dict[str, list[str]] | None = None)`: Stores alias registry reference. Logger: `"soc_db.dedup"`.
       
       - `canonical_id(self, vendor: str, model: str, name: str = "") -> str`: Returns `chip_uuid(vendor, model)` if model is valid (non-empty, contains alphanumeric), else falls back to `slug(name, model)`. **Backward compatibility:** The chip's `id` field remains the slug. UUID goes into a new `uuid` field on the chip dict.

       - `match(self, chip: dict, existing: dict[str, dict]) -> tuple[str | None, str]`: Multi-strategy matching returning (matched_id, strategy_name). Strategies tried in order:

         a) **exact_model** (`"exact_model"`): If chip.get("model") matches existing entry's model (case-insensitive after stripping). Return immediately on match.
         
         b) **alias_registry** (`"alias"`): Normalize chip model via `_normalize_model()` (lowercase, strip non-alphanumeric except spaces/hyphens), look up in alias_registry. If the normalized model is a key, check if any alias matches an existing entry's name or model.
         
         c) **vendor_model_regex** (`"regex"`): Use `extract_model()` from common.py on chip name + model. If the extracted model matches an existing entry's model, return match. Uses per-vendor knowledge patterns if available in VENDOR_KNOWLEDGE.
         
         d) **wikidata_qid** (`"wikidata_qid"`): If chip has `wikidata_id` field, check if any existing entry shares the same `wikidata_id`.
         
         e) **fuzzy_name** (`"fuzzy"`): Use `rapidfuzz.fuzz.token_sort_ratio` with threshold 85 on chip name vs existing entry name. Only consider matches where vendor matches exactly. Return the best match above threshold.

         f) If nothing matches, return `(None, "no_match")`.

       - `_normalize_model(self, model: str) -> str`: Strip whitespace, lowercase, remove non-alphanumeric except hyphens and spaces (for alias key lookup).

       - `batch_match(self, chips: list[dict], existing: dict[str, dict]) -> dict[str, tuple[str | None, str]]`: Run match() for each chip, return dict of original chip id -> (matched_id, strategy).

       - `deduplicate(self, chips: list[dict], existing: dict[str, dict]) -> list[dict]`: Like batch_match but returns only chips that had no match (no match found), plus any matched existing records that should be updated (strategy != "no_match" but new data exists). Each returned chip dict includes a `_dedup_strategy` key showing what happened.

    5. `PRESERVED_ALIASES` constant: Set of ~20 known aliases derived from VENDOR_KNOWLEDGE process_map/gpu_map keys combined with common marketing name patterns. Documented inline.

    Guards:
    - `rapidfuzz` import is tried; if ImportError, the fuzzy strategy logs a warning and returns no match (graceful degradation).
    - The `chip["uuid"]` field must never overwrite `chip["id"]`. UUID is stored separately.
