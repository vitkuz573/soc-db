    **src/soc_db/scraping/sources/wikipedia.py** — WikipediaScraper that implements BaseScraper.

    ```python
    class WikipediaScraper(BaseScraper):
        SOURCE_ID = "wikipedia"
        VENDORS = ["Qualcomm", "MediaTek", "Samsung", "HiSilicon", "Google",
                    "Apple", "Rockchip", "Allwinner", "Amlogic", "Nvidia",
                    "TI OMAP", "Intel Atom", "Ingenic", "NXP i.MX"]
        PRIORITY = 30  # Same as ARCHITECTURE.md pattern
        
        RATE_LIMIT_CONFIG = {
            "requests_per_sec": 1.0,  # Wikipedia rate limit: 1 req/s
            "burst": 2,
            "backoff_factor": 2.0,
            "max_retries": 3,
        }
    ```

    Constructor:
    - Sets `self._http = HTTPSource(rate_limiter=self.rate_limiter)` (inherited from BaseScraper).
    - Sets `self._drift = SchemaDriftDetector(threshold=0.8)`.
    - Registers expected fields with drift detector (see below).

    Methods:

    a) `expected_fields(self) -> set[str]`:
    - Returns the set of field names Wikipedia is expected to produce: `{"cores", "architecture", "gpu", "process_nm", "year", "model", "codename", "npu", "memory_type", "memory_clock", "memory_bus", "modem", "connectivity_wifi", "connectivity_bluetooth", "display_max", "charging", "storage_type"}`.
    - Also include standard meta-fields: `{"id", "name", "vendor"}`.

    b) `fetch(self) -> str`:
    - Gets list of vendor URLs from WIKI_PAGES dict (imported from `soc_db.scraper_wikipedia`).
    - For each vendor:
      - Calls `self.check_robots(url)` before fetch.
      - Calls `self._http.fetch(url, user_agent=self.user_agent)`.
      - Store raw HTML in `self._raw_pages[vendor]`.
    - No parsing yet — just fetch all pages.

    c) `parse(self, raw: dict[str, str]) -> list[ChipScrapeResult]`:
    - Input is the dict of `{vendor: html_string}` from fetch.
    - For each vendor + HTML:
      - Parse with BeautifulSoup (same logic as existing `scrape_vendor` but extracted).
      - Reuse existing parsing functions from `soc_db.scraper_wikipedia`:
        - `parse_standard_table`
        - `parse_transposed_table`
        - `_is_transposed_table`
        - `extract_chip_name`
        - `is_valid_chip_name`
      - Import these directly from `soc_db.scraper_wikipedia` (the legacy module).
      - Convert each parsed chip dict into a `ChipScrapeResult` with:
        - `name=chip["name"]`
        - `vendor=chip["vendor"]`
        - `model=chip.get("model")`
        - `fields=chip` (all parsed fields including id, year, etc.)
        - `source_id=self.SOURCE_ID`
    - Return the flat list of ChipScrapeResult.

    d) `run(self) -> list[ChipScrapeResult]`:
    - Override the default run to handle multi-vendor fetch:
      1. Call `fetch()` to get raw HTML per vendor.
      2. Call `parse()` to get ChipScrapeResults.
      3. Call `dedup()`.
      4. Check drift: for each vendor's results, call `self._drift.check(self.SOURCE_ID, results)`.
      5. Call `write()`.
    - Return results.

    e) `write(self, results: list[ChipScrapeResult]) -> None`:
    - Group results by vendor.
    - For each vendor, call `write_vendor_file(vendor, [r.fields.dict() for r in vendor_results])`.
    - This uses the same `common.write_vendor_file` as the legacy scraper.

    **Expected fields registration:**
    In `__init__`, after super().__init__():
    ```python
    self._drift.register_expected(self.SOURCE_ID, self.expected_fields())
    ```

    **Import strategy:**
    From legacy `soc_db.scraper_wikipedia`:
    ```python
    from soc_db.scraper_wikipedia import (
        WIKI_PAGES,
        scrape_vendor,
        parse_standard_table,
        parse_transposed_table,
        _is_transposed_table,
        extract_chip_name,
        is_valid_chip_name,
    )
    ```
    This ensures the WikipediaScraper reuses the battle-tested parsing logic.
    Do NOT duplicate any parsing code — import from the legacy module.

    From the framework:
    ```python

    from soc_db.scraping.source import HTTPSource
    from soc_db.scraping.drift import SchemaDriftDetector
    from soc_db.common import write_vendor_file
    ```

    **Logging:**
    ```python

    ```

    **Remove any direct import of `fetch` from common.py** — WikipediaScraper uses HTTPSource.
    **Do NOT call `time.sleep(1)` after fetches** — rate limiting is handled by PerSourceRateLimiter.
